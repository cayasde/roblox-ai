# Code comments (policy)

## When to read this

- You’re about to add a comment and want to confirm it adds value.
- You’re reviewing code and want a consistent standard for comments.

## Rule of thumb

Avoid code comments by default. Prefer clarity in the code:

- good names
- small functions
- explicit contracts (types, validation)
- simple control flow (often via guard clauses)
