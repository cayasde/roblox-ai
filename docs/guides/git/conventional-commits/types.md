# Conventional Commits types

## Common types

- `feat`: new feature (often maps to MINOR)
- `fix`: bug fix (often maps to PATCH)
- `docs`: documentation-only changes
- `refactor`: code change that neither fixes a bug nor adds a feature
- `perf`: performance improvement
- `test`: add or update tests
- `chore`: maintenance tasks (deps, tooling, non-user-facing)
- `style`: formatting changes (no logic change)

## Picking a type

Choose based on the primary intent of the change. If a PR mixes concerns, prefer multiple commits.
