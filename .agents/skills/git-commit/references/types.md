# Commit types

## Common types

- `feat`: new feature
- `fix`: bug fix
- `docs`: documentation-only changes
- `refactor`: code change that neither fixes a bug nor adds a feature
- `perf`: performance improvement
- `test`: add or update tests
- `chore`: maintenance tasks, dependencies, or tooling
- `style`: formatting changes with no logic change

## Picking a type

Choose based on the primary intent of the change. If a change mixes concerns, prefer multiple commits.
